# 云端部署说明 - 无需本地开机，多人接收结果

## 方案一：GitHub Actions（推荐，完全免费）

GitHub Actions 是 GitHub 提供的免费 CI/CD 服务，可以每天定时运行你的爬虫，并通过邮件将 CSV 结果发送给多人。

### 优点
- **完全免费**：每月 2000 分钟运行时长，足够本爬虫使用
- **无需维护服务器**：GitHub 自动托管
- **多人收件**：邮件可以发给多个收件人
- **结果存档**：CSV 文件自动保存 30 天，可随时下载

### 缺点
- GitHub 服务器在国外，访问国内政府网站**可能不稳定**
- 如果网络不通，可考虑方案二（腾讯云函数）

---

### 部署步骤

#### 第 1 步：创建 GitHub 仓库

1. 打开 [github.com](https://github.com)，登录你的账号
2. 点击右上角 **+** -> **New repository**
3. 仓库名称填 `gdszfcgw-crawler`，选择 **Public** 或 **Private**
4. 点击 **Create repository**

#### 第 2 步：上传代码

在本地项目目录中执行：

```bash
# 初始化 git（如果还没做过）
git init

# 添加远程仓库（把下面的用户名改成你的 GitHub 用户名）
git remote add origin https://github.com/你的用户名/gdszfcgw-crawler.git

# 添加所有文件
git add .

# 提交
git commit -m "init: 政府采购爬虫"

# 推送到 GitHub
git push -u origin master
```

> **注意**：`config.py` 中的邮箱密码等敏感信息目前是空的，所以即使仓库是公开的也没有泄露风险。真正的密码会通过 GitHub Secrets 设置，不会出现在代码中。

#### 第 3 步：配置 Secrets（敏感信息）

1. 打开 GitHub 仓库页面
2. 点击 **Settings** -> **Secrets and variables** -> **Actions**
3. 点击 **New repository secret**，逐个添加以下 secret：

| Secret 名称 | 说明 | 示例值 |
|------------|------|--------|
| `SMTP_SERVER` | SMTP 服务器地址 | `smtp.qq.com` |
| `SMTP_PORT` | SMTP 端口 | `465` |
| `SENDER_EMAIL` | 发件人邮箱 | `yourname@qq.com` |
| `SENDER_PASSWORD` | 邮箱授权码（不是登录密码！）| `abcdefghijklmn` |
| `RECEIVER_EMAILS` | 收件人邮箱列表（用逗号分隔）| `boss@company.com, colleague@qq.com` |
| `SERVER_CHAN_KEY` | （可选）Server 酱 SendKey | `SCTxxxxxx` |
| `QYWX_WEBHOOK` | （可选）企业微信机器人 webhook | `https://qyapi.weixin.qq.com/...` |

> **QQ 邮箱授权码获取方式**：QQ 邮箱设置 -> 账户 -> 开启 SMTP 服务 -> 获取授权码

#### 第 4 步：测试运行

1. 进入仓库页面的 **Actions** 标签
2. 左侧选择 **政府采购爬虫 - 每日定时运行**
3. 点击右侧 **Run workflow** 按钮手动触发一次
4. 等待运行完成，检查是否收到邮件

#### 第 5 步：查看定时任务

- 工作流已配置为每天 **北京时间 9:00** 自动运行
- 如果某天没有新公告，邮件仍然会发送，内容显示 "发现新项目: 0 条"
- CSV 文件可在每次运行的 **Artifacts** 中下载

---

### 修改定时时间

如需修改每天运行时间，编辑 `.github/workflows/daily-crawl.yml` 第 7 行的 cron 表达式：

```yaml
# 每天 UTC 时间 1:00 = 北京时间 9:00
- cron: '0 1 * * *'

# 改为 UTC 23:00 = 北京时间次日 7:00
- cron: '0 23 * * *'
```

cron 格式：`分 时 日 月 星期`（UTC 时间）

## 方案二：腾讯云函数（国内网络，更稳定）

如果 GitHub Actions 因网络问题无法稳定访问政府采购网，可使用腾讯云函数（SCF）。

### 优点
- **国内网络**：访问广东省政府采购网速度快、稳定
- **免费额度**：每月前 100 万次调用免费，远超需求
- **无需服务器**：无服务器架构，按量计费

### 部署概要

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **云函数 SCF** -> 新建函数 -> 选择 **Python 3.9**
3. 将 `gdszfcgw.py`、`notifier.py`、`scheduler.py`、`config.py` 的代码复制进去（或打包上传 zip）
4. 在函数配置的**环境变量**中添加邮件相关配置（同方案一的 Secrets）
5. 创建**定时触发器**：`0 0 9 * * * *`（每天 9 点）
6. 修改入口函数为执行 `job()` 函数

详细步骤较复杂，如需要可进一步说明。

## 常见问题

**Q: 邮件发送失败怎么办？**
- 检查 Secrets 中的邮箱授权码是否正确（注意不是登录密码）
- QQ 邮箱需要开启 SMTP 服务
- 查看 GitHub Actions 运行日志中的错误信息

**Q: 爬虫运行成功但没有抓到数据？**
- 可能是 GitHub Actions 服务器 IP 被网站屏蔽
- 尝试方案二（腾讯云函数）解决

**Q: 如何添加更多收件人？**
- 修改 `RECEIVER_EMAILS` secret，用英文逗号分隔多个邮箱

**Q: 可以只在有结果时才发邮件吗？**
- 当前已支持：如果 `count > 0`，CSV 会作为附件发送；如果为 0，邮件中也会告知"发现新项目: 0 条"
